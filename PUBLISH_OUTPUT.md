# Publishing Your Generated Client

This document explains how to publish the **output** generated by `cdd-php` (e.g., your PHP SDK/Client) and keep it synchronized.

## Automating the Sync

When working with a separate server OpenAPI specification, your generated client must be kept up to date. You can set up a GitHub Action to automatically rebuild your `ApiClient.php` whenever the upstream OpenAPI spec updates.

### GitHub Actions Workflow

Create `.github/workflows/update-client.yml` in your target repository:

```yaml
name: Update API Client
on:
  schedule:
    - cron: '0 0 * * *' # Run nightly
  workflow_dispatch:

jobs:
  update-client:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Download upstream OpenAPI
        run: curl -sL https://api.example.com/openapi.json -o openapi.json

      - name: Install cdd-php
        run: composer require --dev offscale/cdd-php

      - name: Re-generate Client
        run: vendor/bin/cdd-php from_openapi -i openapi.json

      - name: Create Pull Request
        uses: peter-evans/create-pull-request@v5
        with:
          commit-message: "chore: update OpenAPI client"
          title: "Update API Client to latest OpenAPI spec"
          branch: "update-api-client"
          body: "Auto-generated PR pulling the latest OpenAPI definition."
```

## Publishing the Generated Client

Once your `ApiClient.php` and associated `Models.php` are generated into their own repository, publish it via Packagist like a standard PHP package:

1. In the generated project folder, ensure there is a valid `composer.json` containing `autoload`.
2. Push to GitHub.
3. Submit to [Packagist.org](https://packagist.org/).
4. Developers can now consume your generated API with `composer require your-org/your-api-client`.
