# Usage

The `cdd-php` tool can be used directly as a CLI or embedded within another PHP application as a library.

## Command Line Interface (CLI)

You can run `php bin/cdd-php --help` to get a complete list of commands.

### Converting from OpenAPI to PHP

```bash
# Generate a complete client SDK
cdd-php from_openapi to_sdk -i specs/api.json -o ./my-client

# Generate an SDK CLI
cdd-php from_openapi to_sdk_cli -i specs/api.json -o ./my-cli

# Generate a server skeleton
cdd-php from_openapi to_server -i specs/api.json -o ./my-server
```

### Converting from PHP to OpenAPI

```bash
# Parse a specific file or directory
cdd-php to_openapi -f src/ApiController.php -o openapi.json
```

### Two-way Sync

If you have a directory previously generated by `cdd-php` (or structured similarly), you can synchronize modifications:

```bash
cdd-php sync -d ./my-client
```

This will read your models, routes, controllers, CLI files, and mocks, regenerate the `openapi.json` spec, and overwrite dependent generated files to reflect the unified state.

## Programmatic Library Usage

If you installed `cdd-php` via composer, you can invoke the parsers and emitters programmatically:

```php
<?php
require_once 'vendor/autoload.php';

// Parse a raw OpenAPI string into the intermediate representation array
$openapiSpec = \Cdd\Openapi\parse(file_get_contents('openapi.json'));

// Emit a client SDK
\Cdd\Openapi\emit($openapiSpec, './output-directory');

// Parse a PHP routing file to an OpenAPI paths array
$paths = \Cdd\Routes\parse(file_get_contents('src/routes.php'));
```
